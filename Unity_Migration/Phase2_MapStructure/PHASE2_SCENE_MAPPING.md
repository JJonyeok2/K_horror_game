## Prefab Hierarchy

### Player Prefab

```text
PF_Player
+-  CharacterController
+-  UnityPlayerController
+-  PlayerInteractor input references
+-  MainCamera
|   +-  Camera
|   +-  AudioListener
|   +-  PlayerInteractor ray source
|   +-  Flashlight
|   |   +-  Light(type Spot)
|   +-  HeldItemMounts
|       +-  LeftHandHeldMount
|       +-  RightHandHeldMount
|       +-  TwoHandHeldMount
```

Components:

- `CharacterController`: radius `0.35`, height `1.8`, center y `0.9`.
- `UnityPlayerController`: movement, sprint, stamina, jump, inventory, drop logic.
- `PlayerInteractor`: camera raycast distance `2.4` to `2.7`, layer `Interactable`.
- `Light(Spot)`: range `13`, spot angle `38`, cold flashlight color.
- No `Rigidbody` on the player. Unity `CharacterController` owns motion.

### Artifact Prefab

```text
PF_Artifact
+-  MeshRenderer / MeshFilter
+-  BoxCollider
+-  ArtifactPickup
+-  optional Rigidbody
```

Components:

- `BoxCollider`: default `(0.45, 0.30, 0.45)`, resized per artifact.
- `ArtifactPickup`: display name, value, weight, resentment gain, tags, hand slots.
- `Rigidbody`: only for dropped artifacts. Use dynamic body with freeze rotation if physical tumbling becomes noisy.
- Scene-placed artifacts can be static or kinematic without Rigidbody.

### Extraction Zone Prefab

```text
PF_ExtractionZone
+-  BoxCollider(isTrigger)
+-  ExtractionZone
+-  optional visual floor decal
```

Components:

- `BoxCollider`: size `(2.8, 2.1, 2.4)` for the van interior return zone.
- `ExtractionZone`: calls `GameLoopController.ExtractPlayerInventory()`.
- No `Rigidbody`; player trigger overlap works through `CharacterController` movement.

## Collider And Rigidbody Rules

- Floors: static `BoxCollider`, no Rigidbody.
- Large walls and route blockers: static `BoxCollider`, no Rigidbody.
- Roofs and ceilings: static `BoxCollider` when reachable or needed for occlusion/containment.
- Van structural shell: static `BoxCollider`, no Rigidbody.
- Gate leaves and sliding doors: `BoxCollider`; animate transforms directly. Use no Rigidbody unless physics pushing is intentionally added later.
- Pickup artifacts: `BoxCollider`. Add `Rigidbody` only after the item is dropped into the world.
- Trigger volumes: `BoxCollider(isTrigger)`, no Rigidbody; player `CharacterController` can trigger them.
- Ghost/threat actors in later phases: `NavMeshAgent` + `CapsuleCollider`; Rigidbody optional and usually not needed.

## Coordinate Notes

- Both engines use Y-up, so most positions can be ported directly.
- Godot gameplay forward often uses `-Z`; Unity object forward is `+Z`. Preserve authored positions first, then adjust local mesh/door/camera forward orientation per prefab.
- Door and gate pivots must be authored at hinge transforms, not mesh centers.
- Keep map root pivots at world origin and group offsets under section roots to avoid floating transform drift.
